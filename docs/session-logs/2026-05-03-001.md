## Technical Debt Accumulated

- **Visitor-ID hashing in `WidgetConversation` deferred.** Current rows store raw browser-generated UUIDs in plaintext. **Why it's debt:** weakens our LFPDPPP / data-retention story for Mexico market. **Done right** = SHA-256 hash at insert + lookup, schema migration for existing rows, careful test coverage of the dual-path lookup. Belongs in its own focused PR — bundling it into PR #29 would have made review harder.
- **Output guard only on embed `generateText`, not on authenticated `streamText`.** **Why it's debt:** prompt-leak attack surface is asymmetric (anonymous attackers more likely to probe the embed widget) but not zero. **Done right** = transform stream that buffers a sliding window, runs sentinel checks per chunk, aborts + emits safe fallback on match. Vercel AI SDK supports this via `experimental_transform`. ~1-2 day project.
- **In-memory rate limiter still silently degrades.** `lib/rate-limit.ts` falls back to a per-instance `Map` when `UPSTASH_REDIS_REST_URL` is missing. PR #29 verified Upstash IS set in production, but the silent-degradation behavior is a footgun. **Done right** = fail closed in production if Redis is unavailable; add a startup health check.
- **Hardcoded portion of system prompt still requires a deploy to change.** `lib/ai/prompts.ts` ships with the static prompt. **Why it's debt:** voice/tone/copy tweaks shouldn't need a code release. **Done right** = `SystemPromptVersion` table loaded at runtime, admin UI for review/rollout. ~2-3 day project.
- **CRON_SECRET initially shipped with trailing newline whitespace.** First Vercel preview build failed because `node -e "console.log(...)"` adds `\n`. **Why it's debt:** had to remove + re-add the secret across 3 environments. **Done right** = always use `process.stdout.write` (no newline) when piping secrets into `vercel env add`. Worth memorializing as a memory entry.

## Post-merge regressions caught + fixed (PR #31)

After PR #29 merged, two issues surfaced and were patched on the cleanup branch:

1. **Onboarding Playwright suite started failing.** PR #29 added rate limits to `/api/onboarding/{progress,complete}` (30/min and 10/min per IP). The Playwright onboarding suite makes many sequential calls from the same CI runner IP; with Upstash Redis not configured in CI, the in-memory limiter persisted across the suite and exhausted the per-IP budget mid-run, breaking the wizard step-3 → step-4 transition test. **Fix:** `lib/rate-limit.ts` now short-circuits to `null` (allow) when `isTestEnvironment` is true, mirroring the existing pattern used for the chat message-limit check. Production behavior unchanged.

2. **GitHub Dependabot reported 1 critical vuln still on main** despite my local `pnpm audit --prod` showing 0. Root cause: `@clerk/testing@2.0.8` (devDependency) was transitively pulling vulnerable `@clerk/shared@4.4.0`. `pnpm audit --prod` skips devDependencies; GitHub's Dependabot does not. **Fix:** bumped `@clerk/testing` 2.0.8 → 2.0.24. All `@clerk/shared` instances (prod + dev) now resolve to 4.9.0.

## Lessons / Surprises

- **`vercel env add` from a piped value preserves trailing newlines.** `node -e "console.log(...)"` outputs `<value>\n` which became part of the env var, breaking `Authorization: Bearer <secret>` header construction. Use `process.stdout.write(...)` instead, or `printf` (no newline).
- **`pnpm audit --prod` skips devDependencies.** GitHub Dependabot does not. When evaluating "are we vuln-free for launch", run `pnpm audit` (no `--prod`) too, or rely on GitHub's count. Cost: thought I had 0 critical for launch; main showed 1 critical until the dev-dep bump landed.
- **Adding rate limiting to authenticated routes that have Playwright coverage requires test-env bypass.** The pattern was already in `(chat)/api/chat/route.ts` for the message-limit check; I should have spotted that and applied it to `lib/rate-limit.ts` proactively when adding the rate limits in PR #29. Fixed now at the helper level so all rate-limited routes benefit automatically.
- **`next.config.ts` headers AND `proxy.ts` headers both apply.** When both set the same security header, the LAST one to write it wins (proxy.ts runs after `next.config.ts`'s `headers()`). But if proxy.ts CONDITIONALLY sets a header (e.g., skipping for embed routes), the value from `next.config.ts` carries through. **Single source of truth wins** — moved all security headers to `proxy.ts` in this PR.
- **The Vercel AI SDK has `streamText({ maxRetries: 3 })` built in.** No need to wrap streaming calls in custom retry logic. Only single-shot `generateText` benefits from external retry wrappers.
- **Removing langchain saved 33% of total vulnerabilities in one commit.** When auditing dependency vulns, always check whether the dep is actually USED before bumping it. Many "transitive vulns" are in libs we never imported.
- **`@clerk/localizations` and `@clerk/nextjs` both pulled in the vulnerable `@clerk/shared@4.4.0`.** Bumping `@clerk/nextjs` alone wasn't enough — had to also bump `@clerk/localizations` to fully clear the critical CVE. Always re-audit after each individual bump.
- **The Messenger bot architecture review (provided mid-session by Robert) is high quality.** It does what most architecture docs don't: names the soft spots. Adopted that posture for our own docs going forward (`docs/ARCHITECTURE_LEVERAGE_ANALYSIS.md` follows the same pattern).
- **Pre-existing Playwright failures on `main` had been failing for ~1 month** (confirmed by checking April 2026 main runs). The 4 failures (`admin-contacts/knowledge/settings` unauthenticated tests + health endpoint) are CI environment issues (missing OpenAI embedding model access on the CI's OpenAI project, env validation gaps) — not code regressions. Carried through unchanged on PR #29.
- **Mexican Professional Spanish standard added to global CLAUDE.md mid-session.** Caught my use of `vale` (Iberian) in Spanish router patterns; fixed in this follow-up. Going forward: when writing Spanish text, default to Mexican forms and explicitly avoid the Iberian markers listed in `~/CLAUDE.md`.
